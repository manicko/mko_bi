# BI Dashboard System — Specification

## 1. Overview

Система представляет собой веб-приложение для:

* загрузки пользовательских данных
* обработки и агрегации
* отображения в виде дашбордов
* разграничения доступа по пользователям

Система ориентирована на:

* до 100 пользователей
* до 20 дашбордов
* до 20 графиков на дашборд

---

## 2. Technology Stack

### 2.1 Core

* Python (основной язык)
* FastAPI (backend API)
* Dash (визуализация дашбордов)

### 2.2 Data Processing

* Polars (обработка данных)
* Pgre (хранение агрегатов)

### 2.3 Validation & Models

* Pydantic (валидация данных, схемы API)

### 2.4 Visualization

* Plotly (графики)

### 2.5 Auth & Security

* JWT-based authentication (или аналог)
* bcrypt (хеширование паролей)

### 2.6 Logging

* стандартный logging из Python
* уровни:

  * INFO
  * WARNING
  * ERROR

### 2.7 Dependency Management

* uv (управление зависимостями и окружением)

### 2.8 Testing

* pytest (юнит и интеграционные тесты)

---

## 3. User Roles

### 3.1 Admin

* Создаёт и редактирует дашборды
* Определяет:

  * структуру данных
  * логику обработки
  * набор графиков
* Управляет пользователями
* Назначает доступ к дашбордам

### 3.2 Editor (Data User)

* Загружает данные
* Обновляет данные дашборда
* Не может менять структуру дашборда

### 3.3 Viewer

* Просматривает доступные дашборды
* Не может изменять данные

---

## 4. Authentication

* Логин: email
* Пароль: хранится в хешированном виде
* Пользователей создаёт администратор
* Доступ к дашбордам:

  * через привязку user ↔ dashboard

---

## 5. Data Upload

### 5.1 Формат

* CSV в gzip (.csv.gz)
* Максимальный размер: 10 MB

### 5.2 Поведение

* файл загружается пользователем
* после обработки удаляется
* история не хранится

---

## 6. Data Processing

### 6.1 Общие принципы

* каждая загрузка → полный пересчёт данных дашборда
* обработка выполняется через Polars

### 6.2 Особенности

* структура данных может отличаться для каждого дашборда
* логика обработки задаётся администратором

### 6.3 Типы вычислений

* агрегации
* year-over-year (YoY)
* доли
* кастомные метрики

---

## 7. Data Storage

### 7.1 Принципы

* хранятся только агрегированные данные
* сырые данные не сохраняются

### 7.2 Формат хранения

* DuckDB или Parquet

### 7.3 Организация

* отдельное хранилище на дашборд
* данные общие (не user-specific)

---

## 8. Dashboards

### 8.1 Структура

* дашборды фиксированы (определяются админом)
* каждый дашборд:

  * отдельная страница

### 8.2 Ограничения

* до 20 графиков на дашборд

### 8.3 Графики

* строятся через Plotly
* поддерживают:

  * multi-axis
  * комбинированные графики
  * YoY сравнения

---

## 9. Filters

* глобальные фильтры:

  * год
  * категория
  * бренд
* фильтры применяются ко всем графикам дашборда
* реализуются через backend (Python/SQL)

---

## 10. UI/UX

* минималистичный, но визуально привлекательный интерфейс
* без сложной навигации
* обязательные элементы:

  * страница логина
  * список доступных дашбордов
  * страницы дашбордов

---

## 11. Architecture

### 11.1 Backend

* FastAPI
* отвечает за:

  * авторизацию
  * загрузку файлов
  * управление пользователями
  * доступ к данным

### 11.2 Dashboard Layer

* Dash (встраивается в backend)

### 11.3 Data Layer

* Polars → обработка
* Pgre → хранение

---

## 12. Data Flow

1. Пользователь загружает файл
2. Backend принимает файл
3. Запускается обработка (Polars)
4. Считаются агрегаты
5. Сохраняются в хранилище
6. Дашборд читает агрегаты
7. Строятся графики (Plotly)

---

## 13. Access Control

* пользователь видит только доступные дашборды
* проверка доступа на backend уровне
* каждый запрос к дашборду валидируется

---

## 14. Non-Functional Requirements

* все компоненты бесплатные и open-source
* система разворачивается на одном сервере
* поддержка до:

  * 100 пользователей
  * 20 дашбордов
* время отклика UI: < 1 сек (на агрегированных данных)

---

## 15. Testing & Quality

* покрытие ключевой логики тестами (pytest)
* тестируются:

  * API endpoints
  * обработка данных
  * авторизация
* логирование всех критических операций:

  * загрузка файла
  * ошибки обработки
  * доступ к дашбордам

---
